# requirements.txt 패키지 관리 및 의존성(Dependency) 전략
# ================================================================================
#
# Q1. requirements.txt 파일은 왜 필요한가요?
# A1. '환경의 재현성(Reproducibility)'을 보장하기 위함입니다.
#     팀에 새로 합류한 신규 입사자나 CI/CD 서버(Jenkins 등)가 이 프레임워크를 실행하려면, 
#     우리가 사용한 외부 라이브러리(Selenium, Pytest 등)가 동일하게 설치되어 있어야 합니다.
#     이 파일이 있으면 터미널에 `pip install -r requirements.txt` 명령어 한 줄만 쳐서 
#     완벽하게 똑같은 테스트 환경을 1분 만에 구축할 수 있습니다.
#
# Q2. 패키지 버전 지정 시 `>=` (이상) 기호와 `==` (정확히 일치) 기호의 차이는 무엇인가요?
# A2. 유연성과 안정성의 차이입니다.
#     - `>=` (최소 버전 보장): 라이브러리의 보안 패치나 마이너 업데이트를 유연하게 수용할 수 있습니다. 주로 개발 초기나 템플릿 환경에서 씁니다.
#     - `==` (버전 고정, Pinning): 실제 배포 및 CI/CD 파이프라인에서 가장 권장되는 방식입니다. 
#       어느 날 갑자기 라이브러리가 메이저 업데이트되면서 기존 문법이 바뀌어 멀쩡하던 자동화 코드가 
#       전부 에러를 뿜는 참사(Breaking Change)를 막기 위해, 검증된 특정 버전으로 완전히 못 박아두는 것입니다.
#
# Q3. pytest-html이 아니라 allure-pytest를 쓰는 이유는 무엇인가요?
# A3. 시각적이고 전문적인 '리포팅 대시보드'를 구성하기 위함입니다.
#     단순한 HTML 리포트를 넘어, Allure는 테스트의 트렌드(성공률 변화), 스텝(Step)별 실행 내역, 
#     실패 시 자동으로 캡처된 스크린샷 등을 한눈에 볼 수 있는 엔터프라이즈급 UI를 제공하므로 
#     현업 QA 팀에서 표준으로 가장 많이 채택하는 도구입니다.
# ================================================================================
